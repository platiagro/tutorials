Este componente utiliza a biblioteca AutoKeras

Espera-se como entrada para o componente um arquivo .zip contendo as pastas "train", "test" e "val". Estas três pastas devem conter uma pasta por classe existente no conjunto de dados com as imagens da respectiva classe que serão utilizadas para treino, teste e validação.

Além disso, deve-se fornecer na raiz do arquivo .zip um arquivo dataset.csv, contendo as colunas "image_path", "target" e "subset", onde:

  • image_path: caminho para o arquivo de imagem.
  • target: resposta esperada da predição, caso exista.
  • subset: conjunto ao qual a amostra faz parte, pode ser "train", "test", e "val".

A seguir são listados todos os parâmetros utilizados pelo componente:

  • num_epochs: integer, padrão: 16.
    Número de épocas para executar em cada tentativa de configuração. Deve ser maior ou igual a 2.
  • trials: integer, padrão: 5.
    Quantidade de tentativas de configuração a ser testada.
  • batch_size: integer, padrão: 8.
    Quantas imagens serão enviadas por vez para a rede. Números altos consomem muita memória RAM
  • target_size: integer, padrão: 256.
    Tamanho da imagem de entrada das redes. A imagem será quadrada.
  • brightness_range: float, padrão: None.
    Faixa para escolher o valor da mudança de brilho.
  • channel_shift_range: float, padrão: 0.0.
    Faixa para mudanças aleatórias dos canais.
  • cval: float, padrão: 0.0.
    Valor usado para pontos fora dos limites quando fill_mode = "constant"
  • data_format: string, {["channels_first", "channels_last"]}, padrão: channels_last.
    Formato dos dados da imagem.
  • dtype: string, padrão: float32.
    Tipo para usar nos arrays gerados.
  • featurewise_center: bool, padrão: False.
    Faz a média igual a zero em todo o dataset em termos de features.
  • featurewise_std_normalization: bool, padrão: False.
    Divide entradas pelo desvio padrão (std) do dataset em termos de features.
  • fill_mode: string, {["constant", "nearest", "reflect", "wrap"]}, padrão: nearest.
    Algoritmo usado para preencher regiões caso necessário.
  • horizontal_flip: bool, padrão: False.
    Aumentação] Aplica inversão horizontal nas entragas aleatoriamente.
  • preprocessing_function: string, padrão: None.
    Funções que serão aplicadas em cada entrada. A função irá rodar após a imagem ser aumentada e redimensionada.
  • rescale: float, padrão: 1./255.
    Fator de re-escala. Se None ou 0, nenhuma re-escala será aplicada, caso contrário cada data será multiplicado pelo valor especificado.
  • rotation_range: integer, padrão: 0.
    Faixa de graus para rotações aletórias.
  • samplewise_center: bool, padrão: False.
    Define cada média de amostra para 0.
  • samplewise_std_normalization: bool, padrão: False.
    Divide cada entrada por seu desvio padrão.
  • shear_range: float, padrão: 0.0.
    Intensidade de corte (Ângulo de corte na direção anti-horária em graus).
  • vertical_flip: bool, padrão: False.
    Aplica inversão vertical nas entragas aleatoriamente.
  • zca_whitening: bool, padrão: False.
    Aplica clareamento ZCA.
  • zca_epsilon: float, padrão: 1e-6.
    Valor de epsilon para clareamento ZCA
  • zoom_range: float, padrão: 0.0.
    Faixa para zoom aleatório. Float ou intervalo do tipo [menor, maior].
  • height_shift_range: float, padrão: 0.0.
    Float, 1-D array-like ou integer.
  • width_shift_range: float, padrão: 0.0.
    Float, 1-D array-like ou integer

O retorno durante a experimentação ajuda o usuário observar se o melhor modelo encontrado pelo AutoKeras desempenhou bem através do relatório de classificação e da matriz de confusão obtidos usando o conjunto de teste.

Os retornos são o nome da classe predita e a probabilidade associada a predição.
